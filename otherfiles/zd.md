## 无障碍声道守护

一个修复 Android 13+ 上无障碍播报从耳机跳到扬声器外放的 APP。插着耳机用微信、抖音之后读屏声音窜出来的，可以试试。

下载地址：https://update.wd02.com/a11y-audio-guard/app-release.apk

---

### 这是个什么问题

昨天早上好奇，为什么插着耳机的时候，读屏的无障碍声道有时候会从手机里窜出来，于是就趁它窜出来的时机，抓了一段 ADB 日志。

日志显示，国内很多大厂应用存在乱用音频 API 的情况。具体来说，这些应用在播放语音消息等场景下调用了 `setSpeakerphoneOn(true)`，本意是把听筒切到扬声器外放，但存在两个问题：

1. **用完不释放。** 播放结束后不调用 `setSpeakerphoneOn(false)` 或 `clearCommunicationDevice()`，残留的 `CommunicationRouteClient` 一直挂在系统里。
2. **不判断耳机状态。** 插着耳机的时候音频本来就不走听筒，这个调用完全多余，但它们无条件就调了。

残留的通信路由设置会影响系统 `AudioPolicyManager` 对 `STRATEGY_ACCESSIBILITY` 的路由决策——本来无障碍音频流应该跟着物理输出设备走耳机，现在被这个残留设置带到了扬声器。而且诡异的是，媒体音还在耳机里，只有无障碍播报跑到了外放，不仔细听还以为是自己幻觉。

目前已确认的触发场景包括：微信播放语音消息、播放视频、朋友圈刷视频，抖音播放语音消息，美团 APP 点击视频选项卡甚至在首页什么都不动，豆包结束视频通话等等。除此之外，很多大厂应用的媒体控件可能都存在类似问题。但需要说明的是，触发这个诡异 bug 的路径看起来不止 `setSpeakerphoneOn` 这一条，目前已验证的是这条最常见的，如果后续发现其他成因，也会跟进迭代。

### APP 怎么解决的

原理不复杂：你戴着耳机的情况下，一旦有应用把通信设备切到了扬声器或听筒，APP 通过 `OnCommunicationDeviceChangedListener` 回调捕获这个变化，然后调用 `setCommunicationDevice(headset)` 把通信设备设回耳机。

不戴耳机的时候，应用就安静待着。等你插入耳机后，`AudioDeviceCallback` 回调会唤醒检测逻辑，如果发现通信设备还停留在内建扬声器上，就执行一次修复。

修复之后还有一个 6 秒的恢复观察窗口（500ms 轮询），防止刚修完又被抢回去。如果连续 3 次检测都稳定在耳机上，就退出观察。平时不轮询，不费电。

所有用到的 API 都是 Android 13+（API 33）的公开接口，不需要 root，不需要无障碍权限。

### 关于增强守护模式

APP 里有一个"增强守护"开关，**默认关闭，一般情况下不建议打开。**

它的原理是主动申请 `MODE_IN_COMMUNICATION`（通信模式），让系统认为当前存在一个通信会话，从而加强对耳机路由的控制力。检测到被劫持时，它不会像普通模式那样直接抢，而是先释放本应用的通信设备占用，给系统 200ms 观察它能不能自己恢复到耳机，恢复不了再强制接管。打电话的时候会自动暂停，挂断后自动恢复，不影响正常通话。

但这个模式的副作用是：开启后系统会认为你一直处于通信状态，**可能导致部分应用的音频行为异常**，比如有些应用会检测到通信模式而改变自己的音频策略。所以除非你发现普通模式守不住、声道反复被抢，否则不要开。

### 怎么用

1. 安装后打开，把"启用声道守护"的开关打开
2. 按照提示授权相关权限（电池优化白名单、通知权限等，APP 内有引导）
3. 可以在快捷设置面板添加"声道修复"磁贴，随时手动触发一次修复
4. 然后就不用管了，后台自动守护

支持有线耳机、USB 耳机、蓝牙耳机（含 BLE）。

### 当前状态

我自己使用了一天，感觉效果还是可以的。但我手里只有一台破红米，发给过几个朋友使用，不过大家大概率都是破红米，在破红米上稳定性挺好的，其他手机咱就不知道了。

如果有人想要试试的话，下载地址在最上面，用着有问题的多顶顶帖、提个 bug，或者直接在 GitHub 上提 issue 都行，看看效果到底怎么样，值不值得将它继续更新下去。毕竟它值得完善的地方还是很多的，比如说无障碍声道被扬声器抢走这件事，成因不止这一个；再比如说既然实现了对音频路由的调整，是不是可以增加更多的玩法，一切都还不知道。

我起初只是好奇这个问题，让 AI 写了一段 Java 来复现，然后想着把这个逻辑做成 APP 挂在后台，帮我修声道，没想到它的复杂性以及未来可能有的机遇会不小，那就发出来大家一起乐呵乐呵吧。
